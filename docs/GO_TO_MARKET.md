## 1. 商业化的一句话

**Solo Compass 不是一款"通用旅行 App"，而是一款"为独自旅行者设计的思考工具"。** 它的定价必须反映"工具"的心理定价 anchor，而不是"服务"的月费 anchor。

三条判断：

1. **不做订阅焦虑**。目标用户是数字游民 / 高知独行者，最讨厌"又一个每月扣款的 App"。
2. **不做免费主义**。免费的旅行 App = Google Maps / 小红书。Solo Compass 的价值是"付费换取无广告、无推荐操纵、隐私自持"，免费会自毁定位。
3. **给用户"一次决定"的机会**。买断 = 我认可这个工具，年费 = 我在这个季节需要它。两种心理都要接住。

## 2. 定价方案（锁定）

### 双轨结构

| Plan            | 价格       | 包含                                     | 心理锚点                       |
| --------------- | ---------- | ---------------------------------------- | ------------------------------ |
| **Free**        | $0         | 城市地图、5 个体验/天、基础 Solo-Score    | Try-before-install（Scenario B）|
| **Pro Yearly**  | $50 / year | 全部功能 + 云端同步 + 未来所有更新        | "这一年我会独自旅行"           |
| **Pro Lifetime**| $29 一次性 | 全部功能 + 云端同步 + 当前所有功能永久    | "我认可这个工具，一次买清"     |

> **为什么 Lifetime 反而更便宜**：这不是错。Lifetime = 现有功能锁死买断，未来重大新功能进入 Yearly 池。参考 Craft / Fastmail 的做法。Lifetime 覆盖"个人使用心理成本极低 → 传播意愿高"的独立开发者、极客用户；Yearly 覆盖"我要最新的 AI 交叉编译、最新的 Ritual、云同步保障"的深度用户。两者不冲突，反而互相导流。

### Pro 解锁什么（对比表 SoT）

功能名与 iOS 端代码对齐（见 `apps/ios/SoloCompass/`）：

| 类别      | 功能                                            | Free            | Pro                     |
| --------- | ----------------------------------------------- | --------------- | ----------------------- |
| 地图      | 城市地图、Nearby、Filter                        | ✅              | ✅                      |
| AI 交叉编译| Multi-source enrichment、Reasoning trace         | 每日 3 次       | 无限                    |
| 路线      | 单城市当日路线 (RoutesSection)                   | ✅              | ✅                      |
| 路线      | 跨城市 + AI 生成个性化路线                       | ❌              | ✅                      |
| Rituals   | Time Capsule、Daily Omen、Blindbox               | 只读            | 创建 & 收藏             |
| Companion | 好友系统、约伴、Realtime 聊天                    | ✅              | ✅                      |
| 隐私      | On-device only                                  | ✅              | ✅                      |
| 同步      | 跨设备 iCloud + Supabase                        | ❌              | ✅                      |
| 打印      | Print Partner (导出杂志式旅记)                   | ❌              | ✅                      |
| Live      | 灵动岛 + 锁屏 Live Activity                      | 每日 3 次       | 无限                    |
| 未来      | 所有新 App 内 AI 模型升级                       | 半年后开放       | 首发                    |

> **反 anti-pattern**：Free 版本从不"限速"、不弹"升级提示广告"。限制是**次数上限**，不是**体验降级**——降级会污染品牌。用户到达上限时，看到的是一次性的、可 dismiss 的、写着"你今天做完了 3 次 AI 咨询，明天见 · 或者解锁 Pro"的暖琥珀卡片，不是弹窗。

### 家庭 / 团购

Phase 1 **不做**。理由：独行者产品做家庭共享是身份撕裂。等 Phase 2 有 100+ 家庭反馈再上。

### 学生 / 教育折扣

Yearly 半价 = $25。凭 `.edu` 邮箱或 UNiDAYS 验证。**首发不做**，Phase 1.5 上线。

### 地区定价（重要）

App Store 国区（CNY）不能直接 6.99 汇率换算——那样会显得"贵国内产品好几倍"。采用**心理价** anchoring：

| 地区          | Yearly              | Lifetime            |
| ------------- | ------------------- | ------------------- |
| US            | $50                 | $29                 |
| CN            | ¥198                | ¥118                |
| JP            | ¥7,800              | ¥4,500              |
| EU            | €48                 | €28                 |
| Southeast Asia| Apple auto (低 40%) | Apple auto (低 40%) |

日本 ARPU 极高（旅行文化 + 独居文化），价格锚要往上抬。东南亚新兴市场靠 Apple 自动汇率。

## 3. 目标用户（POV 用户画像）

不是"25-40 岁数字游民"这种宽泛画像。三个真实场景：

### Persona A: "清迈游民" Nikita

- 32 岁乌克兰籍女性远程工程师，护照压力大，在清迈住 3 个月
- iPhone 15 Pro，付费 App 池: Craft ($50/yr)、1Password ($36/yr)、Bear ($30/yr)、Fastmail ($60/yr)
- 每天 4 pm 开始"晚上去哪"焦虑：不想去太吵的地方，不想连续 3 天同一家咖啡
- **决策路径**：Reddit `/r/digitalnomad` 看到分享 → 官网 → Free 试 3 天 → 买 Lifetime（$29 心理无痛，且"我一次性支持独立开发者"符合她的价值观）

### Persona B: "东京独居" Kentaro

- 38 岁日籍男性，本地人，周末独自去箱根 / 镰仓
- Apple One + 20+ 付费 App，习惯为"细腻工具"付费
- 讨厌 Google Maps 的"推荐给你"，觉得**推荐系统在操纵他**
- **决策路径**：App Store JP 编辑推荐 → 官网 → 直接 Yearly ¥7,800（因为他每年独自出行 15+ 次，年费更划算 + 想要最新功能）

### Persona C: "国内高知" 陆瑶

- 29 岁上海产品经理，一年独自去 3-4 次云南 / 大理 / 松阳
- 深度小红书用户但已疲劳；ChatGPT Plus 用户
- **反感"什么都好"的模糊推荐**，想要"就是这家咖啡因为它下午 3 点没人 + 靠窗 + 提供英文书"
- **决策路径**：小红书 KOL 独行博主种草 → 官网中文版 → Free 试 → 买 Lifetime ¥118（决策心理: "还不如一顿饭钱"）

三种场景合起来给出了官网必须解决的**三个诚意问题**：

1. **给 Nikita**: "证明你是独立开发者，不是又一个 VC 割韭菜产品"
2. **给 Kentaro**: "证明你不做推荐操纵"
3. **给陆瑶**: "证明你的推荐颗粒度是店铺级 + 具体时间段，不是 city guide 级"

## 4. 竞品切位与差异化叙事

不比"功能对比"，比"心智对位"。

| 竞品          | 他们的心智                     | 我们相对他们的一句话                                                           |
| ------------- | ------------------------------ | ------------------------------------------------------------------------------ |
| Google Maps   | "全世界的地址簿"                | 我们只放"值得独自一个人去"的地方——1000 个 vs 100 万个                          |
| 小红书        | "看别人玩什么"                  | 你在看别人，我们让你成为主角——没有社交、没有 like、没有比较                    |
| ChatGPT       | "会说话的百科"                  | 我们的每条建议都能追溯到具体来源和最后核实时间——AI 不虚构                     |
| Airbnb Trips  | "打包卖你完整行程"              | 我们卖你"下一步"，不卖你"整个星期"——留出你自己的空白                            |
| Lonely Planet | "权威静态指南"                  | 我们的"下午 3 点+雨天+周三"是动态的——静态权威已死                              |
| TripAdvisor   | "情侣/家庭的评论海"             | 我们是唯一一个把"你独自一人"作为默认前提的地图——非家庭视角                     |

**官网 Hero 副标题就用这个矩阵的某一句**（英文版首屏候选）：

> "Not another travel app. A map where every dot is worth a solo detour."

（延续 `PRODUCT_BRIEF.md` 里 "not another travel app" 的立场）

## 5. Launch Sequence

Phase-gated 上线，每阶段一个可衡量指标：

### Phase 1 — Soft Launch (Week 0–4)

- 官网上线，中英两版
- App Store US + JP + CN 首发
- **单一 KPI**: 30 天内 500 个 Free download，30% 试用 Pro 功能
- 渠道: 独立开发者社区（Product Hunt / IndieHackers / V2EX 独立项目板）
- **不做付费广告**。付费投放前必须有自然口碑基础

### Phase 2 — Content-driven Growth (Month 2–6)

- SEO 长尾内容开始积累（见 `SEO_STRATEGY.md`）
- KOL 合作: 3 位真实独行博主的产品植入（小红书 / YouTube）
- **单一 KPI**: 有机搜索流量 > 1000 UV/月，转化率 ≥ 3%
- 引入 Lifetime "早鸟" $19 促销（限首 1000 名）

### Phase 3 — Category Definition (Month 6+)

- 目标：让"solo travel companion app"这个搜索词的第一个 App Store 结果就是我们
- 内容护城河：城市专题页 100+，Blog 长文 30+
- **单一 KPI**: MRR > $10k（约 200 个 Yearly + 300 个 Lifetime）

## 6. 商业化红线（Anti-goals）

延续 `PRODUCT_BRIEF.md` 的"至少和它做的一样重要"的反面清单：

- ❌ **不做广告**。任何广告都在毁 solo travel 的心境。
- ❌ **不做数据变现**。用户位置数据 on-device only 是核心承诺。见 `PRIVACY.md` 4 表。
- ❌ **不做分发抽成**。永远不接第三方"合作商家"抽成——一旦收钱就不再客观。
- ❌ **不做限时促销制造焦虑**。买断价可以随时降，但不做"48 小时倒计时"这种反 solo 心境的营销手段。
- ❌ **不做 in-app purchase 蚕食**。除了 Free/Yearly/Lifetime 三档，没有第四种付费点。
- ❌ **不做订阅套餐让人算不明白**。永远只有两个 Pro 档位 + 一个 Free。

如果任何商业化提案违反上述任一条，PR 描述里必须显式说明"为什么这次值得违反"。

## 7. 官网 → App Store 转化模型

三层漏斗，每层都有明确要求：

```
访问官网 (Web)
    ↓ Hero 屏说服 → 30% 继续滚动
Features / Trust / Pricing 深度浏览
    ↓ 3 个疑虑消除 (隐私/AI 幻觉/定价) → 8% 点击 CTA
点击 "Get on App Store" 或 "Get on TestFlight"
    ↓ App Store 页面二次说服 → 40% 下载
Free 试用
    ↓ 3 天内触发 Pro 功能 → 6% 付费
成为 Pro
```

综合转化率：**访问 → 付费 ≈ 0.6%**。这是"高价值工具"合理转化，不追求 SaaS 式 3%。

**每一层都要有对应的官网元素**：

| 漏斗层            | 官网职责                                          | 视觉/文案要求                                     |
| ----------------- | ------------------------------------------------- | ------------------------------------------------- |
| 首屏 30% continue | Hero 必须 3 秒内讲清"这是什么 + 为谁做"           | 见 `WEB_LANDING_DESIGN.md` 第 3 节                |
| 疑虑消除 8% CTA   | Trust 区块 (Privacy · AI transparency · Pricing)  | 三个诚意问题（第 3 节）逐个正面回应               |
| App Store 40%     | CTA 前的最后一次证据（screenshot + testimonial）  | 真人使用截图 + 一句话背书（非"5 星好评"套话）      |
| Free → Pro 6%     | 官网无法直接影响；靠 App 内的 Pro 引导设计         | 见 iOS 端的 `Views/Paywall/`（未来）              |

## 8. 收入预测（诚实版）

不做 hockey stick 幻觉，做 base-case:

| 时期   | Free DAU | Pro Users | Yearly% | Lifetime% | MRR   | ARR   |
| ------ | -------- | --------- | ------- | --------- | ----- | ----- |
| M1     | 200      | 30        | 30%     | 70%       | $138  | $1.6k |
| M3     | 800      | 120       | 35%     | 65%       | $525  | $6.3k |
| M6     | 2000     | 300       | 40%     | 60%       | $1400 | $17k  |
| M12    | 5000     | 750       | 45%     | 55%       | $3800 | $46k  |
| M24    | 12000    | 1800      | 50%     | 50%       | $9500 | $114k |

M24 = 独立开发者能"这就是我唯一的工作"的临界点。这是终点，也是新起点（团队扩张 / 团队订阅 / API 授权）。

## 9. 一句话总结（可以背下来的）

> **Solo Compass 不追求成为下一个 Airbnb。它追求成为下一个 Fastmail——独立开发者的、值得付一次钱的、你会告诉朋友的、有立场的工具。**

如果任何商业化决策与这句话冲突，重新审视决策。
