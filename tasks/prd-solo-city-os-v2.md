## 0. 一句话定位(v2 收敛版)

> **一张会呼吸的暖琥珀地图:我一个人已经在这座城,它替我判断"现在、这里、独自去,行不行",并陪我把这一段活明白。**

拆解三个关键词,恰好对应三份来源文稿:

- **"替我判断行不行"** = 信心内核(confidence PRD 的北极星:NowScore × SoloScore × Confidence)
- **"把这一段活明白"** = 城市 OS(city-os PRD 的生命周期容器:落地→立足→生活→离开)
- **"会呼吸的暖琥珀地图"** = 已落地的设计语言(CT token 系统),它不只是皮肤——见 §2,它就是产品语言本身

## 1. 三份文稿的关系与张力消解

先诚实列出三股力量的方向差,再给出裁决。这是本稿最重要的思考,后面所有规格都由此推出。

### 1.1 关系图

```
┌────────────────────────────────────────────────────────┐
│  City OS(外壳)— 城市容器 · 生命周期 · 模块编排           │
│  ┌──────────────────────────────────────────────────┐  │
│  │  信心内核(心脏)— NowScore × SoloScore × Confidence │  │
│  │  三条骨架规则:独行透镜 / 会呼吸 / 信心贯穿一切       │  │
│  └──────────────────────────────────────────────────┘  │
│  暖琥珀设计语言(皮肤兼神经)— 三条规则的可见形态          │
└────────────────────────────────────────────────────────┘
```

**裁决一(聚合 vs 收敛,表面矛盾,实则同构)**:confidence PRD 说"砍到只剩信心层",city-os PRD 说"聚合多模块"。融合点在于——**三条骨架规则就是信心层的泛化**。信心层原本只作用于 Experience;城市 OS 把同一套判断施加到一切聚合信息上:一条 eSIM 信息有 `lastVerifiedAt` 健康度、一场活动有独行友好度、一个街区有此刻安全度。所以不存在"聚合稀释信心"的问题,只存在"聚合内容是否过了三关"的问题。**聚合是信心层的扩张,不是它的对立面。**

**裁决二(V_NEXT 的情绪仪式 vs 反目标)**:V_NEXT_DESIGN 推 Blindbox/Capsule/Omen 等情绪峰值功能,confidence PRD 判它们为 gamification 反目标。融合裁决用一条硬标准:**仪式只有在强化三条骨架规则之一时才允许存活**(详见 §7.2)。设计系统里已为它们建好 token(`capsuleGlow`/`omenGold`/`blindboxAmber`),说明视觉投资已发生——裁决结果是"重定位后保留三个、原样砍零个",而非全砍。

**裁决三(社交面)**:两份 PRD 一致:第 4 层(归属/社交)规避。现状 `FeatureFlags.companion` Release 默认 OFF 已符合。37 个 Companion 文件不删库,但**唯一保留进主线的产出是路线分享卡**(`docs/design/route-map-share-card.md`)——它是病毒传播物,不是社交图谱。Connect 模块按 city-os 规格纯外链。

**裁决四(V_NEXT 的"砍 BottomInfoSheet 中间层")**:暂缓。R0 冷启动核心 UX 依赖 peek 档(已有 #131 推迟决议);v2 的做法是**先把 peek 升级为「此刻卡片」**(§5.1),让中间层配得上它占的位置,再评估是否收敛为两档。

## 2. 融合洞察:设计语言即产品语言

这是本次融合最核心的发现。检视已落地的 `CompareTokens.swift`,三条骨架规则在色彩系统里已经有**1:1 的语义映射**——只是从未被显式声明过:

| 骨架规则(city-os §2)         | 色彩语义(CT token)                                                        | 已上线的组件族                                                                       |
| ----------------------------- | -------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| **① 独行透镜** Solo Lens      | 暖琥珀 `accent 0x5D3000` / `accentSoft` — 独行底气的颜色                    | SoloScoreCard heatmap 维度条 · hint 行 · 問Solo dock 入口 · 主 amber pill            |
| **② 会呼吸** Living/时间感知  | 此刻金 `sunGold 0xC9A677` / `sunGoldSoft` / `sunGoldDeep` — 时间窗口的颜色  | BestTimeRibbon 好窗口 band · peek「此刻最值得去」header · 灵动岛 IslandPalette       |
| **③ 信心贯穿** Confidence     | 核验绿 `verifiedGreen` + 健康度点 🟢🟡🔴⚫ + TrustBadge 五桶                  | trustChip 三态 · TrustBadge(verified/amap/osm/userCreated/curated)· L{n}·信号 mono 行 |

**由此推出 v2 的第一设计公理**:

> 任何新模块(落地包、在地、合规……)接入时,**不发明新颜色语义**。凡表达"独自去没问题"用琥珀,凡表达"此刻是窗口"用金,凡表达"信息可信度"用健康度点 + TrustBadge。用户在详情页学会的视觉语法,在落地包里零成本复用。

这直接回答了 city-os PRD 最担心的"链接目录陷阱":目录型 App 的每个模块各长一张脸;我们的每个模块说同一种视觉方言——**一致的语法本身就是"过了三关"的可见证明**。

第二公理:**mono 字体 = 可证伪的数据**。`CT.mono` 已用于 `L{n} · {n} 信号`、ETA、坐标、based-on 计数。规则化:凡是"诚实暴露的原始数字"(信号数、汇率、签证剩余天数、183 天计时)一律 mono——用户潜意识里学会"mono 的东西是可查证的",这就是"诚实、可证伪"人格的字体级表达。

## 3. 统一体验模型:一屏 · 一卡 · 一岛

v2 的全部体验收敛为三个表面,各司其职:

```
一屏(地图)      = 空间感知:哪里值得去。Marker 由 NowScore 驱动呼吸。
一卡(此刻卡片)   = 决策:现在去行不行。北极星三行答案,BottomInfoSheet peek 常驻。
一岛(灵动岛)     = 时间感知:锁屏/岛上流动的"此刻"。日落窗口、出发倒计时、落地欢迎。
```

- **屏**回答 *where*,**卡**回答 *now what*,**岛**回答 *when*。三者共用同一套 token(`CT` ↔ `IP` 镜像已存在)。
- 灵动岛是"会呼吸"规则的**平台级护城河**——Nomad List 是网页,Google Maps 不做独行时间窗。已有 7 个 Kind(route/countdown/recording/compile/soloAgentHint/timeCapsule/dailyOmen),v2 只做接线与重定位,不加新 Kind。
- Chat 不是第四表面:它已按 confidence PRD 降级为意图输入,产出物是卡片(`ChatCard`/`RouteProposal`)落回屏与卡,`AgentStatusLine` → `ReasoningSummaryChip` 的折叠链路已上线,方向正确,保持。

## 4. 信息架构 v2

### 4.1 根状态:城市 + 模式 + 阶段

沿用 city-os §3,根状态 = `(city, mode, phase)`:

- `mode`: **Live**(在此城·此刻)/ **Plan**(未到·计划)/ **Recall**(已离·回顾)
- `phase`: 抵达 Land → 立足 Settle → 生活 Live → 离开 Leave(驱动模块浮现)

**顶部城市栏**(升级现有城市切换):`displayRounded` 城市名 + 模式徽章。Plan 模式整屏罩一层 `washLight` 暖白纱(token 已有)表示"预演态";Recall 模式城市名旁加 `sunGoldDeep` 的"回顾"标。**模式切换 = 整个聚合上下文切换**,不是筛选器。

### 4.2 BottomInfoSheet = 生命周期货架

三档 detent 不变(peek/mid/full,固定高度+offset 平移的丝滑机制保留),但**mid/full 的内容从"固定栏目"改为"按 phase 浮现的货架"**:

| 阶段       | mid 档货架(自上而下)                                          | 备注                                        |
| ---------- | ------------------------------------------------------------- | ------------------------------------------- |
| 抵达 Land  | 落地包卡(可收起)→ Now shelf → Nearby                          | 落地包首次主动浮现,之后收进货架首位薄条     |
| 立足 Settle| Now shelf → 能干活的地方(咖啡/work 切片)→ Nearby              | 立足 = 第 2 层需求,轻做,复用现有分类切片   |
| 生活 Live  | Now shelf → Routes → 在地本周(§5.3)→ Nearby                   | 现有主形态 + 新增在地流                     |
| 离开 Leave | 回顾卡(本段图鉴 + 留下独行者信号 CTA)→ Now shelf              | Recall 贡献引擎入口                         |

- **peek 档永远是此刻卡片**(§5.1),不随 phase 变——北极星常驻。
- 货架浮现规则就是"生命周期驱动、不做 30 格图标墙"的落地形式;FilterBar 保持信心切片方向(Now / Solo-friendly / Trusted),不为新模块加 chip。

### 4.3 情境横幅与打扰预算

合规提醒(签证/税务临界)、安全预警(在地新闻里影响出行的)以**横幅**形式插在货架顶,用 `warningSoft/warningText`(token 已有)。硬预算:**同屏最多 1 条横幅,每日主动浮现总次数 ≤ 2**——"不做 engagement loop"反目标的量化形式。

## 5. 核心界面规格(暖琥珀版)

### 5.1 此刻卡片 Now Card —— peek 档的北极星形态

confidence PRD §6.4 的三行结构 × CT token 渲染。升级现有 `PeekSummaryCard`:

```
┌─────────────────────────────────────────────┐  surfaceWhite 卡 · borderSubtle
│ ● 此刻最值得去                    ⌁ sunGold  │  ← peekHeaderLabel(已有)
│ ☕ Ristr8to        现在 87% 好时机  sunGoldDeep│  ← 行1 NowScore:displayRounded 标题
│    日落还有 23 分钟 · 晴 · 人少               │     + reason(fgMuted)
│ 🧍 Solo 8.5 「坐吧台,楼上更安静」   accent    │  ← 行2 SoloScore:heatmap 微条 + hint
│ 🟢 3 位独行者本周去过 · L3 · 5 信号  mono     │  ← 行3 Confidence:健康点+TrustBadge+mono
│ [ 带我去 ]                    [ 换一个 ]      │  ← 主 amber pill + ghost 按钮
└─────────────────────────────────────────────┘
```

- 行 1/2/3 分别使用金/琥珀/绿三语义——**用户每天看几十次的这张卡,就是三条骨架规则的每日教学**。
- 「换一个」永远给下一个 option,绝不空手——"AI 只筛选不决定"的按钮级表达。
- basedOnCount = 0 时行 3 显示「AI 估算 · 待验证」(灰),诚实优先。
- 点卡 → mid 档展开;点「带我去」→ 导航;长按 → 加入行程。零新 sheet。

### 5.2 落地包 City Ready-Kit —— 暖琥珀薄卡组

city-os §4 的四项 + 在地预告,渲染规则:

- 容器:`surfaceSunken 0xF3EEE6` 分组底 + 每行 `surfaceWhite` 薄卡,结构与 SoloScoreCard 同语法。
- 每行右侧:健康度点 + `lastVerifiedAt` 相对时间(mono)。**签证剩余天数 / 183 天计时用 mono 大数字**,这是本模块唯一自算、离线可用的部分,视觉上要配得上它的可靠性。
- 独行透镜提示行(「机场先用 eSIM,别柜台换卡」)用 `accentSoft` 底 + `accent` 字——与详情页 hint 行同款,语法复用。
- 外链行为:深链跳出前底部 toast「已为你打开 Airalo · 回来继续」,回链感知。急救号码卡**离线常驻**、一键拨号,不跳出。

### 5.3 在地 Live 模块 —— 时间轴流

差异化模块(Nomad List 没有),规格:

- 形态:mid 档货架里一节「本周在这座城」,横滑卡组;full 档展开为按天分组的纵向流。
- 每张活动卡 = 缩略图 + `displayRounded` 标题 + **独行友好徽章**(SoloScore 思路施加到事件:琥珀 `Solo-OK` chip + 一句 hint「一个人逛正好,傍晚人最舒服」)+ 时间窗口(`sunGoldSoft` band,BestTimeRibbon 微缩版)+ 健康度点(活动信息极易过期,⚫ 直接置灰)。
- **回流地图**:活动落图成限时 Experience,marker 带金色时限环——在地与信心地图打通,这是"聚合内容过三关"的样板。
- `dailyOmen` 灵动岛 Kind 重定位为**「今日城市签」= 在地模块的每日一条摘要**(今天这座城最值得独自去做的一件事),每日 1 条上限已有(`solo.liveactivity.count` 机制)。

### 5.4 详情页(已达标,做基准)

暖琥珀 DayPage 重做(hero 信任 chip / heatmap 评分条 / BestTimeRibbon / 暖琥珀 dock / 問Solo / 旅人共建层)**已经是 v2 语法的完成态**——它就是其他所有界面的对齐基准。唯一增量:Recall 模式下 dock 主按钮从「标记完成」换为「留下独行者信号」(评分 + 一句话 hint 贡献),接通冷启动数据引擎。

### 5.5 意图输入(Chat 降级完成态)

保持现有卡片化链路。补一条约束:意图结果**优先以地图筛选高亮 + Now Card 置换**呈现,`ChatCard` 流仅在多候选需要对比时出现。`ReasoningTracePanel` 是"AI 解释自己"的差异化资产,保留但默认折叠为 `ReasoningSummaryChip`。

## 9. Roadmap 融合(单一时间线)

> 原两份 PRD 各有 R1–R3,合并为一条线,信心内核先行——心脏没跳,外壳无意义。

**R1 · 收敛 + 内核可见(2–3 周)** = confidence R1+R2
- 旗标关闭第 4 层入口(现状已大半完成,验收即可);仪式层按 §7.2 重定位或下线。
- NowScore 补 Crowd/SafetyTime 信号 → marker 呼吸;此刻卡片上线(§5.1)。
- Gate:任一时段有三高点 demo;30 秒可用性测试说得清产品;`parity:check` 绿。

**R2 · 城市容器 + 单城打透(3–4 周)** = city-os R1
- `(city, mode, phase)` 状态机 + 生命周期货架;落地包四项(人工策展版)+ 在地模块(人工策展版)在清迈或万象一座城上线。
- 灵动岛 dailyOmen 接在地摘要。
- Gate:一个真实用户靠它"落地→立足→探索"走完一天不跳去别的 App。

**R3 · 聚合成型 + 打磨(3–4 周)** = city-os R2 + confidence R3
- 落地包外链化 + 签证税务自算;在地接活动源 + 独行友好标注;三条骨架规则审查所有模块。
- 排序压制 level≤1、离线 fallback、性能门全过。
- Gate:需要者周 2 留存 ≥ 40%;跨 App 替代率有正向信号。

## 10. 成功指标(合并去重)

- **主指标**:完成率(推荐→真去做的 %)· 需要者周 2 留存 ≥ 40%。
- **聚合指标**:每次到访的跨 App 替代率(本来要开的 N 个游民 App 被吸收几个)。
- **过程指标**:地图→决策步数(≤2)· 此刻卡片「带我去」点击率 · 在地卡完成率。
- **反指标**:社交/聊天停留时长回升 = 漂回第 4 层;自身停留极短且无判断被采纳 = 目录陷阱;横幅日均 >2 = 打扰预算破产;盲盒复购与三高供给脱钩 = 变成老虎机。
