# PRD: Solo City OS v1 — 独行者城市生活操作系统(聚合体架构)

| 字段     | 值                                                                   |
| -------- | -------------------------------------------------------------------- |
| 版本     | v1.0 草稿                                                            |
| 创建日期 | 2026-07-05                                                           |
| 方向决策 | **聚合/超级 App**:多工具聚合为主体,信心地图为核心模块之一            |
| 范围     | apps/ios/SoloCompass 为主                                            |
| 关联     | 信心地图沿用 `prd-solo-confidence-refactor-v1.md`(降为 Explore 模块) |
| 前置事实 | 市场无成功「超级 App」;Nomads.com 已占「决策去哪座城」的聚合位       |

---

## 0. 一句话定位

> **不是"决定去哪座城"的聚合器(那是 Nomad List),而是"我一个人已经在这座城、帮我把这一段活明白"的城市生活 OS。**

聚合是形态,**独行 + 在地 + 此刻**是灵魂。没有后者,它就是又一个链接目录,会死。

---

## 1. 决策背景与两条铁律

用户已明确选择聚合方向。但落地前必须承认两条市场铁律,架构据此设计:

1. **没有任何"数字游民超级 App"跑通** — 生态刻意碎片化,专用工具各赢一块。→ 推论:我们不重造工具,我们**编排 + 加判断**。
2. **Nomads.com 已是那个聚合体** — 1000+ 城市成本/网速/安全排名、10000+ 社区、183 天税务计时、气候查找、行程追踪。→ 推论:**不能正面拼广度与数据量**,必须换一个它做不到的轴。

### 换轴:Nomad List vs Solo City OS

| 维度     | Nomads.com(要避开的正面战场) | Solo City OS(我们的楔子)         |
| -------- | ---------------------------- | -------------------------------- |
| 使用时机 | **行前**:决定去哪座城        | **在地**:已经到了,怎么活         |
| 粒度     | 城市级排名(静态)             | 街道级 · 时间感知(会呼吸)        |
| 视角     | 通用远程工作者               | **独自一人**的尴尬/安全/底气     |
| 形态     | 数据榜单 + 社区              | **地图为家** + 情境聚合          |
| 核心问题 | "该去哪座城?"                | "我一个人,现在,在这座城,怎么办?" |

**同一批用户,不同的一段人生阶段。** 不抢它的饭,接它的棒。

---

## 2. 聚合体里怎么保住护城河(最关键一节)

纯聚合的死法:不拥有数据、不产生判断、外链一堆 = 无护城河。破解 = 三条贯穿所有模块的"骨架规则",让 Nomad List 和随手做的目录都抄不动:

**① 独行透镜(Solo Lens)** — 聚合来的通货**不原样展示**,一律穿过独行视角再表达。
不是"WiFi 50Mbps",是"这家能干活、独行友好、日落前都安静";不是"某活动今晚 8 点",是"这个活动**适合一个人去**,不会尴尬"。

**② 会呼吸(Living / 时间感知)** — 所有聚合内容都带**此刻/那时**维度(沿用 NowScore 的 `at:` 投影)。Nomad List 是静态榜单,你是随时间和到访阶段变化的活体。

**③ 信心贯穿一切(Confidence on everything)** — 不只地图上的 Experience,连"这条 eSIM 信息/这个签证规则"也带 `lastVerifiedAt` 与健康度点 🟢🟡🔴⚫(复用 `confidence.ts`)。**"诚实、可证伪"从地图延伸到整个聚合体**,这是信任型产品的统一人格。

> 三条规则 = 聚合体的"操作系统内核"。任何模块接进来,都必须过这三关,否则就是掉进链接目录陷阱的信号。

---

## 3. 架构:以「城市」为容器,以「到访生命周期」为主线

### 3.1 城市即上下文(呼应"按状态探索不同城市")

整个 App 的根状态是**当前探索的城市 + 模式**,而非 GPS。切换城市 = 切换整个聚合上下文。三种模式(承接上一轮讨论):

- **在此城 · 此刻(Live)** — GPS 命中已收录城市,默认态。聚合模块全部实时。
- **未到 · 计划(Plan)** — 选一座未来城市 + 投影时段。落地包变"行前清单",地图变"预演"。
- **已离 · 回顾(Recall)** — 已标记去过。你从消费者转为**贡献者**,反哺信心数据(即冷启动数据引擎)。

### 3.2 生命周期驱动的模块浮现(不是 30 格图标墙)

聚合**不做常驻工具面板**。模块按"到访阶段"自动浮现——这是与目录型 App 的根本区别:

```
抵达 ──────► 立足 ──────► 生活 ──────► 离开
Land        Settle       Live         Leave
落地包        找落脚         信心地图+在地      回顾/贡献
(联网/钱/    (住/能干活的    (探索此刻该做      (图鉴/留下
 签证/安全)    地方/通勤)       的事 · 本地活动)     独行者信号)
```

### 3.3 模块清单

| 模块               | 内容                                                     | 处理方式               | 是否护城河   |
| ------------------ | -------------------------------------------------------- | ---------------------- | ------------ |
| **① 落地包 Land**  | 联网 · 钱 · 签证/税务天数 · 安全/急救(见 §4)             | 编排 + 外链 + 独行透镜 | 中(体验)     |
| **② 立足 Settle**  | 住(coliving/青旅)· 能干活的地方 · 通勤                   | 外链 + SoloScore       | 中           |
| **③ 探索 Explore** | **信心地图**:此刻该做的事(NowScore×SoloScore×Confidence) | **自建**               | **强(核)**   |
| **④ 在地 Live**    | 本地活动/节日/新闻(用户新增需求,详见 §5)                 | 聚合 + 独行透镜        | **强(差异)** |
| **⑤ 合规 Comply**  | 183 天税务计时 · 签证到期提醒 · 签证规则                 | 自算 + 外链            | 低(必要)     |
| **⑥ 连接 Connect** | 认识人:Meetup/青旅/本地群                                | **纯外链,不自建**      | 无(第4层)    |

> ③ 与 ④ 是护城河所在,资源优先。①②⑤⑥ 是"接人进城 + 留人"的必要基建,尽量外链、轻做。

## 4. 落地包(City Ready-Kit)规格 — 用户选定的四项优先

抵达/选中城市时弹出,一次性、可收起、可随时从地图抽屉再唤出。每行一张薄卡,点击深链到外部 App;信息带健康度点。

```
🛬 落地包 · 万象 Vientiane
├ 📶 联网      Airalo 老挝 eSIM ₭ · 或 Unitel 门店       🟢 本月核验  →
│              「机场到市区先用 eSIM,别机场柜台换卡」(独行透镜提示)
├ 💰 钱        Wise 汇率最优 · ATM 每笔手续费高,建议一次多取  🟡 →
│              「多数咖啡馆只收现金;BCEL One 本地扫码」
├ 🛂 签证/税务  落地签 30 天 · 已停留 3 天 · 距 183 天税务线 180 天  🟢
│              [开启到期提醒]
├ 🛡 安全      市中心/湄公河岸独自较稳 · 夜间避开偏僻河堤  →GeoSure
│              🆘 191 警察 · 1195 旅游警察 · 使馆电话
└ 🎉 在地      本周:那伽火球节前导市集(周五)· 适合一个人逛  →见「在地」
```

四项数据来源:

- **联网**:eSIM 深链(Airalo/GigSky 等)+ 本地运营商门店 POI;网速可选接 Speedtest/众包。
- **钱**:汇率 API(Wise/开放汇率)+ "本地支付习惯"人工/AI 策展一句话。
- **签证/税务天数**:**自算**——本地记录入境日,183 天计时(抄 Nomads.com 已验证的刚需),签证规则库外链/AI 摘要。
- **安全**:GeoSure/官方旅行提示深链 + 急救号码本地库(离线可用)。

---

## 5. 「在地」模块(Live)— 用户新增,且是真差异化

Nomad List 不做"本周这座城有什么值得一个人去的活动/节日/新闻",这块是空的,且高留存。

- **内容**:本地活动、节日、市集、展览、本地新闻(影响出行的:罢工/封路/天气预警)。
- **独行透镜过滤**:每个活动标"独自友好度"——一个人去会不会尴尬、要不要结伴、几点人最舒服。→ 直接复用 `SoloScore` 思路到"事件"上。
- **时间感知**:以"本周/今天"为轴的在地流,不是静态列表。
- **来源**:本地活动平台/票务(Eventbrite/Meetup 类)深链 + 节日日历 + 城市新闻源;冷启动阶段可 AI 聚合 + 人工策展前几条(与种子城市策略一致)。
- **回流地图**:活动落到地图上成为限时 Experience,信心地图与在地模块打通。

---

## 6. 首页与信息架构

- **首页 = 信心地图(Explore 模块)**。聚合模块以顶部"城市 + 阶段"栏 + 底部/侧边抽屉的形式**从属于**地图,绝不另起竞争首页。
- 落地包在"抵达/切城"时**主动浮现一次**,之后收进抽屉。
- 合规提醒(签证/税务)以**情境横幅**在临界时出现,平时不打扰。
- 保持 `docs/PRODUCT_BRIEF.md` 反目标底线:聚合**不等于**做 social feed / 打卡积分。连接他人一律外链。

## 7. 数据来源与工程策略

| 类型                               | 策略                                          | 成本 |
| ---------------------------------- | --------------------------------------------- | ---- |
| 通货信息(网速/汇率/签证/活动/天气) | **外链深链 + 少量 API**,不自建、不存权威库    | 低   |
| 签证/税务天数                      | 本地计算 + 本地存储(隐私友好)                 | 低   |
| 信心地图 Experience                | 沿用现有 AI 合成 + 策展管线(唯一要深耕的数据) | 高   |
| 急救号码/安全区                    | 小型离线本地库,按城市维护                     | 低   |

要点:**除信心地图外,几乎不承担"拥有数据"的成本**——聚合模块靠编排,把稀缺工程资源全押在护城河(③④)上。这也直接回答了最初的"数据受限/冷启动"焦虑:大部分模块本就不需要你拥有数据。

## 8. 分期 Roadmap

**R1 · 内核 + 单城打透(3–4 周)**

- 落地城市容器 + 三模式(Live/Plan/Recall)状态机。
- 信心地图作为 Explore 模块跑通(即 confidence-refactor PRD 的 R1–R2)。
- 万象或清迈一座城,落地包四项 + 在地模块**人工策展**上线。
- Gate:一个真实用户能靠它"落地→立足→探索"走完一天,不跳去别的 App。

**R2 · 聚合成型(3–4 周)**

- 落地包外链化(eSIM/Wise/GeoSure 深链)+ 签证税务自算计时。
- 在地模块接活动源 + 独行友好度标注。
- 三条骨架规则(独行透镜/会呼吸/信心)审查所有模块。
- Gate:需要者周 2 留存 ≥ 40%;"打开本 App 而非打开 N 个 App"的主观反馈。

**R3 · 多城 + 打磨(持续)**

- 城市生命周期自动浮现调优;第 2、3 座城市复制;性能与离线。

---

## 9. 成功指标

- **主指标** — 每次到访的"跨 App 替代率":用户本来要开的 N 个游民 App,有几个被本 App 吸收/替代。这是聚合体存在的唯一理由。
- **主指标** — 需要者周 2 留存 ≥ 40%。
- **护城河指标** — Explore/在地模块使用时长与完成率(独行判断被采纳)。
- **反指标** — 若产品沦为"外链跳板"、自身停留极短且无判断被采纳,即掉进目录陷阱,预警。

---

## 10. 风险与对策

| 风险                                    | 对策                                                         |
| --------------------------------------- | ------------------------------------------------------------ |
| **变成平庸链接目录,无护城河**(最大风险) | §2 三条骨架规则是硬门槛;每个模块必须过独行透镜/时间/信心三关 |
| 广度稀释深度,资源摊薄                   | ①②⑤⑥ 一律外链轻做;工程只深耕 ③④;单城打透再复制               |
| 正面撞上 Nomads.com                     | 换轴:在地/此刻/独行,不做城市榜单;接它的棒不抢它的饭          |
| 外链体验割裂(跳来跳去)                  | 深链 + 回链设计;关键信息(急救/签证天数)本地化不跳出          |
| 聚合数据维护成本(活动/签证规则易过期)   | 用 `confidence` 健康度暴露过期;宁可标"待核验"也不假装新鲜    |
